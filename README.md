## Executive summary

SRE observability stack with Prometheus, Grafana, OpenTelemetry, SLOs, alerting rules, runbooks and incident review.

**Primary outcome:** Define an observable operating model that improves incident response and reduces alert noise.

**Professional signal:** Shows reliability maturity, SLO thinking, incident review and full-stack observability direction.

This repository is designed to be readable by three audiences:

| Audience | What to look for |
|---|---|
| **Recruiters / ATS** | Clear keyword alignment with cloud infrastructure, platform engineering, SRE, FinOps, DevSecOps, Kubernetes, Terraform and GitHub Actions. |
| **Engineering managers** | Operating model, trade-offs, governance, maturity path, security posture and cost/reliability reasoning. |
| **Hands-on reviewers** | Repository structure, automation patterns, runbooks, validation workflow and implementation roadmap. |
